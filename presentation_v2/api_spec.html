POST /api/predict-risk REQ-F-005 · TC-API-001 위험률 산출
담당: 홍제규
개요
Unity C#에서 현재 타임스텝의 센서값 16채널을 전송하면, 로드된 ML 모델이 위험률(0~100)을 예측하여 반환합니다. 응답 지연 시 이전 risk_score를 유지하는 fallback 처리가 Unity 측에서 구현됩니다.
Request Body (application/json)
필드타입필수설명
sensor_1floatRequired채널 1 센서값 (정규화 전)
sensor_2floatRequired채널 2 센서값
… (3~15)floatRequired채널 3~15 센서값
sensor_16floatRequired채널 16 센서값
scenario_idintOptional시나리오 ID (로그 연동용, 기본값 1)
Response 200 (application/json)
필드타입설명
risk_scorefloat위험률 0.0 ~ 100.0
risk_levelstring정상 / 주의 / 위험 / 차단
thresholdfloat현재 적용 중인 threshold 값
response_msint서버 처리 시간 (ms)
risk_level 기준:  정상 < 40  ·  주의 40~59  ·  위험 60~69  ·  차단 ≥ 70 (threshold)
Request 예시
"sensor_1": 0.42, "sensor_2": 0.87, "sensor_3": 0.31, // ... sensor_4 ~ sensor_15 ... "sensor_16": 0.55, "scenario_id": 1
Response 예시
"risk_score": 64.0, "risk_level": "위험", "threshold": 70.0, "response_ms": 42
GET /api/automation-metrics REQ-F-017 자동화 지표 조회
담당: 홍제규 + 이성룡
개요
차단 지연시간, 차단 시퀀스 완결성, 자동 복구 성공률, 무인 데모 완주 여부 등 자동화 성능 지표를 반환합니다. Unity 대시보드의 자동화 지표 화면에서 사용됩니다.
Response 200 (application/json)
필드타입설명
cutoff_latency_msint위험 감지→밸브 닫힘 지연 (목표: < 1000)
sequence_completebool밸브·경보·환기 3단계 모두 실행 여부
recovery_success_ratefloat자동 복구 성공률 (0.0 ~ 1.0)
unmanned_demobool무인 데모 완주 (사람 개입 0) 여부
thresholdfloat적용 중인 고정 threshold (70)
modelobject위험률 모델 정보 객체
model.namestringRandomForest (얇은 위험률 API)
model.loadedboolmodel.pkl 로드 여부
Response 예시
"cutoff_latency_ms": 420, "sequence_complete": true, "recovery_success_rate": 1.0, "unmanned_demo": true, "threshold": 70.0, "model": { "name": "RandomForest", "loaded": true }
GET /api/scenarios REQ-F-001 시나리오 목록
담당: 홍제규
개요
Unity 시나리오 선택 화면에서 호출합니다. 연구소 / 개발실 / 배관실 3개 시나리오의 설정값(누출 위치, 가스 종류, threshold)을 반환합니다.
Response 200 (application/json)
필드타입설명
scenariosarray시나리오 객체 목록
scenarios[].idint시나리오 고유 ID
scenarios[].namestring시나리오 이름
scenarios[].locationstring누출 발생 위치
scenarios[].gas_typestring가스 종류
scenarios[].thresholdfloat해당 시나리오 기본 threshold
scenarios[].descriptionstring시나리오 설명
Response 예시
"scenarios": [ { "id": 1, "name": "연구소", "location": "배관 연결부 A", "gas_type": "CH4", "threshold": 70.0, "description": "협소 공간 고위험" }, { "id": 2, "name": "개발실", "location": "실험대 측면", "gas_type": "CO", "threshold": 72.0, "description": "환기 가능, 중위험" }, { "id": 3, "name": "배관실", "location": "주배관 밸브", "gas_type": "LPG", "threshold": 65.0, "description": "광역 차단 시나리오" } ]
POST /api/logs/cutoff REQ-F-015 · TC-DB-002 자동차단 이벤트 저장
담당: 홍제규 + 장영재
개요
Unity에서 risk_score ≥ threshold 조건이 충족되어 자동차단이 발생하면 즉시 호출합니다. cutoff_events 테이블에 이벤트를 저장하며, 이후 결과 요약 화면에서 조회됩니다.
Request Body (application/json)
필드타입필수설명
scenario_idintRequired발생 시나리오 ID
cutoff_timestringRequired차단 발생 시각 (ISO 8601)
risk_at_cutofffloatRequired차단 시점의 risk_score
threshold_usedfloatRequired적용된 threshold 값
sim_elapsed_secintOptional시뮬레이션 시작부터 경과 시간(초)
Response 201 (application/json)
필드타입설명
event_idint생성된 cutoff_events PK
statusstring"saved"
Request / Response 예시
// Request "scenario_id": 1, "cutoff_time": "2026-07-13T14:32:05", "risk_at_cutoff": 81.2, "threshold_used": 70.0, "sim_elapsed_sec": 47 // Response 201 "event_id": 12, "status": "saved"
GET /api/health REQ-NF-006 서버 상태 확인
담당: 홍제규
개요
Unity 시뮬레이션 시작 전 FastAPI 서버 및 ML 모델 로드 상태를 확인합니다. model_loaded가 false이면 Unity에서 "서버 연결 실패" 안내를 표시하고 데모 모드(로컬 모델)로 전환합니다.
Response 200 (application/json)
필드타입설명
statusstring"ok" / "degraded"
model_loadedboolML 모델 로드 완료 여부
model_namestring현재 로드된 모델명
db_connectedboolDB 연결 상태
uptime_secint서버 가동 시간(초)
Response 예시
// 정상 "status": "ok", "model_loaded": true, "model_name": "RandomForest", "db_connected": true, "uptime_sec": 342 // 모델 미로드 (degraded) "status": "degraded", "model_loaded": false, "model_name": null, "db_connected": true, "uptime_sec": 5
HTTP 코드에러 타입설명 및 발생 조건
422 Validation Error Request Body 필드 누락 또는 타입 불일치. Pydantic 자동 검증. detail 배열에 필드별 오류 명시.
500 Internal Server Error 모델 predict() 실패, DB 쿼리 오류 등 서버 내부 예외. 로그에 traceback 기록.
503 Service Unavailable 모델 미로드 상태에서 /predict-risk 호출 시 반환. Unity는 fallback 처리(이전 risk_score 유지).
공통 에러 Response 형식
// 422 Validation Error 예시 { "detail": [ { "loc": ["body", "sensor_1"], "msg": "field required", "type": "value_error.missing" } ] } // 500 Internal Server Error 예시 { "detail": "Model prediction failed" } // 503 Service Unavailable 예시 { "detail": "ML model not loaded. Use fallback." }
Unity 에러 처리 원칙 — API 호출 실패(타임아웃, 503) 시 이전 risk_score를 그대로 유지하고 HUD에 [API 연결 중] 표시. 3회 연속 실패 시 로컬 데모 모드로 자동 전환. (REQ-NF-006)
