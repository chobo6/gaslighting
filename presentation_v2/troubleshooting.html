예상 이슈 · 데이터
UCI 데이터셋의 risk_score 라벨 기준 불명확
가스 센서 원본 데이터에는 위험률이 명시되어 있지 않아, 농도·속도·지속시간 기반으로 직접 라벨을 생성해야 함. 라벨 설계 기준이 불명확하면 모델 학습 품질에 영향을 미침.
대응 방향
① 농도(ppm) 임계값 기반 1차 점수 산출 → ② 증가 속도(slope) 가중치 부여 → ③ 지속 시간 패널티 적용 방식으로 rule-based 라벨 생성. 라벨 생성 함수를 명시적으로 문서화하여 재현성 확보.
예상 이슈 · ML
Threshold 탐색 시 오탐률과 차단 시점 오차 트레이드오프
threshold를 낮추면 조기 차단(오탐 증가), 높이면 늦은 차단(위험 누락). 발표용 시연에서 설득력 있는 threshold를 선정해야 함.
대응 방향
F2-score(Recall 가중) 기준으로 threshold 후보(60~95 범위) 스캔. 오탐률 <10%, Recall >0.9 조건을 만족하는 구간에서 차단 시점 오차 최소값 선택. 선정 근거를 모델 성능 화면에 시각화.
예상 이슈 · Unity-API 연동
Unity C#에서 FastAPI 비동기 호출 시 HUD 동기화 지연
1초 단위 HUD 갱신 요건에서, API 응답 지연 발생 시 위험률 표시가 실제 시뮬레이션 타임스텝과 어긋날 수 있음.
대응 방향
UnityWebRequest 비동기 코루틴으로 API 호출. 응답 실패 시 이전 risk_score 유지(fallback). 발표용 데모 모드에서는 로컬 모델 직접 호출로 API 의존성 제거 가능.
예상 이슈 · Unity
Particle System 성능 저하로 시뮬레이션 프레임 드랍
가스 확산 파티클 수가 많아지면 3D 씬 렌더링 성능이 저하되어 HUD 갱신 주기에 영향을 줄 수 있음.
대응 방향
Max Particles 제한 설정 + LOD 적용. 발표용 데모에서는 파티클 품질을 중간으로 고정하여 안정적인 30fps 이상 유지. 발표 환경에서 사전 테스트 필수.
