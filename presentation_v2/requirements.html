REQ-F-001시나리오 선택사용자는 연구소/개발실/배관실 등 시나리오를 선택할 수 있어야 한다.상선택한 시나리오에 따라 누출 위치, 가스 종류, threshold가 초기화된다.Unity
REQ-F-002가스 누출 발생시뮬레이션 시작 시 지정된 지점에서 가스 누출이 발생해야 한다.상누출 시작 시 Particle System과 경고 상태가 화면에 표시된다.Unity
REQ-F-003시간 기반 농도 변화시간 경과에 따라 가스 농도, 센서 반응값, 위험률이 갱신되어야 한다.상1초 단위로 센서값과 위험률 HUD가 갱신된다.Unity
REQ-F-004센서 데이터 재생UCI 가스 센서 시계열 또는 가공 CSV를 Unity 시뮬레이션에 재생할 수 있어야 한다.상CSV 타임스텝별 sensor_1~sensor_16 또는 농도값이 적용된다.Data/Unity
REQ-F-005위험률 회귀 예측Backend 또는 로컬 모델은 현재 센서값을 입력받아 위험률 0~100%를 예측해야 한다.상risk_score가 0~100 범위로 반환된다.ML
REQ-F-006Threshold 고정 선정자동차단 기준 threshold를 규칙 기반 고정값으로 선정해야 한다.상LEL 기준 근거와 함께 threshold가 고정값(70)으로 설정된다.ML/규칙
REQ-F-007위험률 그래프 표시시간별 위험률 곡선과 threshold 기준선을 표시해야 한다.상그래프에서 위험률 곡선이 기준선을 넘는 지점이 시각화된다.Unity UI
REQ-F-008자동차단/복구 이벤트위험률이 threshold 이상이면 자동 차단하고, 안전값 복귀 시 자동 복구해야 한다.상risk ≥ threshold 시 valve CLOSED, 안전 복귀 시 인터락 조건 충족하면 자동 재개방된다.Unity/Control
REQ-F-009경보/환기 작동자동차단 시 경보등, 사이렌, 환기팬이 작동해야 한다.상차단 이벤트 발생 시 alarm_status=ON, ventilation_status=ON으로 표시된다.Unity
REQ-F-010위험 등급 표시위험률에 따라 정상/주의/위험/차단완료 상태를 표시해야 한다.상HUD에 위험 등급과 색상 상태가 표시된다.Unity UI
REQ-F-011위험 원인 설명위험률 상승 원인을 농도, 증가 속도, 지속시간 기준으로 설명해야 한다.중위험 원인 패널에 주요 기여 요인이 표시된다.ML/UI
REQ-F-012결과 요약시뮬레이션 종료 후 최대 위험률, 차단 시점, 최종 상태를 요약해야 한다.상결과 화면에 max_risk, cutoff_time, final_status가 표시된다.Unity UI
REQ-F-013수동 파라미터 조정누출 속도, 환기율, threshold를 사용자가 조정할 수 있어야 한다.중슬라이더 값 변경 후 재실행 시 차단 시점이 달라진다.Unity UI
REQ-F-014예측 API 연동Unity는 현재 센서값을 Backend로 전송하고 위험률 예측 결과를 받을 수 있어야 한다.중POST /api/predict-risk 응답이 HUD에 반영된다.Backend/Unity
REQ-F-015로그 저장센서값, 위험률, 차단 이벤트를 DB 또는 파일에 저장해야 한다.중gas_readings, risk_predictions, cutoff_events에 로그가 저장된다.DB
REQ-F-016데모 모드실제 센서 없이 준비된 데이터로 발표용 시뮬레이션을 재생할 수 있어야 한다.상누출→위험률 상승→자동차단 흐름이 자동 진행된다.Unity
REQ-F-017자동화 지표 조회차단 지연시간, 시퀀스 완결성, 무인 데모 완주 등 자동화 지표를 확인할 수 있어야 한다.중지표 화면에 차단 지연·시퀀스 완결성·자동 복구 성공률이 표시된다.Dashboard

REQ-NF-001사용성비전문가도 위험률과 자동차단 흐름을 이해할 수 있도록 구성해야 한다.상위험률, threshold, 차단 상태가 한 화면에서 확인된다.UX
REQ-NF-002시각성가스 확산, 경보, 밸브 차단이 명확하게 보이도록 해야 한다.상정상/주의/위험/차단 상태가 색상과 애니메이션으로 구분된다.Visual
REQ-NF-003응답성시뮬레이션 중 위험률 예측 및 UI 갱신이 지연되지 않아야 한다.상위험률 HUD가 1초 이하 주기로 갱신된다.Performance
REQ-NF-004재현성동일 시나리오와 seed에서는 동일한 차단 시점을 재현할 수 있어야 한다.중동일 입력 데이터 재생 시 cutoff_time이 동일하다.Data
REQ-NF-005확장성새로운 가스 종류와 공간 시나리오를 추가할 수 있어야 한다.중scenario 설정 데이터 추가만으로 새 데모를 구성할 수 있다.Architecture
REQ-NF-006안정성Backend API가 실패해도 Unity 데모가 중단되지 않아야 한다.상API 실패 시 fallback 로직 또는 오류 안내가 작동한다.Fault Tolerance
REQ-NF-007안전성실제 장치를 직접 제어하는 시스템이 아님을 명확히 표시해야 한다.중문서와 UI에 교육/시뮬레이션 목적 문구가 포함된다.Safety
REQ-NF-008문서화기획 발표에 필요한 산출물을 HTML/MD 형태로 제공해야 한다.상기획서, WBS, 요구사항, ERD, 메뉴트리, 시나리오, 스토리보드가 존재한다.Documentation
